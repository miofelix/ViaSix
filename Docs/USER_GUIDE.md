# ViaSix 用户指南

ViaSix 的主要用途是让 Mac 的公网流量通过一个经过优选的 IPv6 代理入口。它不提供服务器、账号或订阅，你需要使用自己有权访问的 Mihomo 配置。

## 1. 理解 IPv6 链路

ViaSix 的严格链路是：

```text
Mac 应用流量 → TUN → 所选 IPv6 地址 → 远程代理 → 目标网站
```

“IPv6 模式”保证第二段使用 IPv6，即 Mac 到远程代理入口使用所选 IPv6。远程代理访问网站时可能使用 IPv4 或 IPv6，所以出口检测结果不一定是 IPv6。

主界面只保留五个工作区：

- **首页**：查看链路就绪状态、启停连接和检测出口。
- **IPv6 优选**：测试、比较并应用 IPv6 节点。
- **连接配置**：导入、查看和编辑 Mihomo YAML。
- **日志**：查看代理、测速和应用事件。
- **设置**：管理运行组件、TUN 服务、本机参数和应用数据。

## 2. IPv6 模式与兼容模式

### 2.1 IPv6 模式

新安装默认使用此模式，启动需要同时满足：

1. TUN 服务和特权 Mihomo 已就绪；
2. 当前节点是有效 IPv6 地址；
3. 配置包含至少一个可替换 `server` 的内联代理。

运行时 ViaSix 会：

- 只保留主内联代理；
- 把主代理的 `server` 替换为当前 IPv6；
- 固定使用规则模式；
- 让私有、回环和链路本地地址直连；
- 让其余被 TUN 接管的流量匹配主代理；
- 不加载导入的代理组、Provider、规则 Provider、子规则和自定义规则。

严格模式下，系统代理、规则/全局/直连切换和本地代理/TUN 接入选择不会显示，因为这些选项与严格链路冲突。

### 2.2 兼容模式

兼容模式用于旧配置和特殊需求，保留：

- IPv4 测速和 IPv4 节点；
- Provider-only YAML；
- 导入的代理组与规则；
- 规则、全局和直连模式；
- 仅本地 mixed 端口、macOS 系统代理和可选 TUN；
- 运行期间的基础 Selector 代理组切换。

缺少 `ipv6TransportPolicy` 字段的旧 `local-proxy.json` 会迁移为兼容模式，避免升级后改变既有网络行为。

## 3. 首次配置

### 3.1 安装测速组件

打开“设置 → 运行组件”，安装 CloudflareSpeedTest。也可以导入本机已有的兼容可执行文件。

普通用户态 Mihomo 主要供兼容模式的本地代理使用。IPv6 模式使用 TUN 服务管理的固定特权 Mihomo，不会执行设置中的自定义 Mihomo。

### 3.2 准备 TUN

在“设置 → 虚拟网卡服务”依次完成：

1. 安装或批准系统服务；
2. 安装或修复特权 Mihomo；
3. 确认服务、运行时和会话能力正常。

本地 ad-hoc 构建首次安装时会请求管理员授权。日常启动、停止和重连不需要重复授权。

可配置 TUN 网络栈、MTU 和严格路由。通常保留 Mixed、1500 和默认严格路由设置即可；与其他 VPN 冲突时再调整。

### 3.3 导入连接配置

打开“连接配置”，导入 `.yaml` 或 `.yml`。严格模式推荐单个内联 VLESS、VMess、Trojan 或 Shadowsocks 节点。

最小示例：

```yaml
proxies:
  - name: My VLESS
    type: vless
    server: origin.example.com
    port: 443
    uuid: 11111111-1111-1111-1111-111111111111
    network: ws
    tls: true
    servername: origin.example.com
    ws-opts:
      path: /proxy
      headers:
        Host: origin.example.com
```

ViaSix 只替换服务器地址，不改动端口、凭据、传输、TLS/REALITY、SNI、Host 或路径。

高级编辑器仍接受通用 Mihomo YAML。在 IPv6 模式下这些字段可以保存，但运行投影只采用主代理；需要完整执行 Provider、代理组或自定义规则时请切换兼容模式。

## 4. IPv6 优选

1. 打开“IPv6 优选”。
2. 默认选择内置 IPv6 列表；也可以选择自定义文件或输入 IPv6/CIDR。
3. 根据需要调整 TCPing/HTTPing、端口、筛选条件和性能参数。
4. 点击“开始测速”。
5. 按延迟、丢包率和速度比较结果，选择一个候选地址。
6. 点击“应用节点”；如果连接正在运行，ViaSix 会重新连接。

IPv6 模式不会显示内置 IPv4 数据源，也不会允许应用 IPv4 结果。兼容模式会恢复 IPv4 数据源和相关参数。

测速结果只反映候选入口在当前网络下的表现，不等于代理服务器到所有网站的最终速度。

## 5. 启动与检查连接

首页会分别显示：

- TUN 是否就绪；
- 当前 IPv6 是否有效；
- 配置是否支持注入 IPv6；
- 当前连接是否运行。

全部就绪后点击启动。严格模式会创建特权 TUN 会话；停止连接或退出应用时，ViaSix 只停止当前用户自己启动的会话。

出口 IP 卡片用于观察最终网站出口，并不证明客户端入口地址族。判断 IPv6 入口是否生效，应以首页的 IPv6 链路状态、当前节点和日志为准。

兼容模式运行后，首页可显示 Mihomo 中可手动选择的 Selector 代理组。ViaSix 只在启动或用户刷新/选择时读取这一状态，不持续采集连接、流量或内存数据。

## 6. 日志

日志是长期保留的独立界面，可用于检查：

- 应用启动和配置同步；
- IPv6 节点应用与重连；
- TUN 服务、会话和恢复；
- Mihomo 启停和错误输出；
- CloudflareSpeedTest 进度与失败原因；
- 出口检测结果。

界面支持按来源和级别筛选、跟随最新记录、改变排序方向和清空当前会话日志。清空只影响界面中的当前内存记录，不修改代理配置或测速结果。

## 7. 兼容模式的网络接入

在“设置 → 本机代理”切换到兼容模式后，可以选择：

- **仅本地代理**：应用手动使用 `127.0.0.1:11451` mixed 端口；
- **系统代理**：ViaSix 在连接运行期间配置 macOS HTTP、HTTPS 和 SOCKS 代理，并在停止或退出时恢复；
- **TUN**：接管不遵循系统代理的流量；
- **系统代理 + TUN**：两个入口可同时启用。

兼容模式还可使用规则、全局和直连路由。直连模式不需要服务器配置；其他模式需要内联代理或 Provider。

## 8. 数据、备份与恢复

默认数据目录：

```text
~/Library/Application Support/ViaSix/
```

主要文件：

- `Data/preferences.json`：节点、测速和应用偏好；
- `Data/profile.yaml`：用户连接配置，可能包含敏感凭据；
- `Data/local-proxy.json`：IPv6/兼容策略、本机监听和 TUN 参数；
- `Data/Mihomo/config.yaml`：可重新生成的用户态运行配置；
- `Data/result.csv`：最近一次成功测速结果；
- `Logs/`：组件日志目录。

备份和恢复前应从菜单栏完全退出 ViaSix。不要在应用运行时覆盖这些文件。

## 9. 常见问题排查

### 提示“需要先选择有效的 IPv6 节点”

打开“IPv6 优选”并应用一个 IPv6 地址。自定义源可能包含 IPv4，严格模式不会应用它。

### 提示“当前配置无法注入 IPv6 节点”

当前 YAML 可能只有 Provider，或没有可替换 `server` 的内联代理。改用内联配置，或切换兼容模式。

### 提示“需要先准备虚拟网卡服务”

进入“设置 → 虚拟网卡服务”，检查系统批准、特权 Mihomo 和 helper 功能。如果另一个用户占用会话，需要由会话所有者停止。

### 最终出口是 IPv4

这是允许的。ViaSix 的严格保证是本机到代理入口使用 IPv6，不保证代理服务器到目标网站的出口地址族。

### 与 VPN 或其他网络工具冲突

先停止 ViaSix 和其他 TUN/VPN 工具，再单独启动 ViaSix。必要时调整 TUN 网络栈或严格路由，并查看日志中的路由、DNS 和恢复信息。

## 10. 安全建议

- 只使用自己有权访问的代理配置和测速目标。
- 不要公开 `profile.yaml` 或包含凭据的日志片段。
- 优先使用维护者签名并公证的正式应用。
- 不要关闭 Gatekeeper 或系统完整性保护来绕过安装问题。
- 完全退出应用后再删除数据目录或恢复备份。
